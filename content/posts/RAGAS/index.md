## RAGAS 思想

**RAGAS** (Retrieval Augmented Generation Assessment) 是一套 **用 LLM 评估 RAG 系统** 的自动化评估框架，由 Exploding Gradients 团队在 2023 年提出。

### 核心思想

传统评估 RAG 管线需要人工标注标准答案，成本高、周期长。RAGAS 的核心思想是 **LLM-as-a-judge** —— 用大模型充当评分员，对 RAG 输出的各个维度自动打分，无需人工标注。

它的假设是：**LLM 能够判断一段文本是否忠实于上下文、是否切题、上下文是否相关**。基于这个假设，设计了一系列 Prompt 模板 + 评分公式，把定性判断转化为 0~1 的定量分数。

------

## 关键指标及实现逻辑

### 1. Faithfulness（忠实度）

**衡量的核心问题**：答案有没有胡说八道？每个陈述是否都能在检索到的上下文中找到依据？

**实现流程**：

```
① 拆陈：将答案拆成原子性陈述句（每个表达一个独立事实）
   例: "万用表可以测电压。红色表笔插VΩ孔。" → ["万用表可以测电压", "红色表笔插VΩ孔"]
   
② 验证：对每个陈述，用 LLM 判断"这个陈述是否能被上下文直接支持"
    Prompt: "判断以下陈述是否可以被上下文中的信息直接支持？只回答 true/false"
    
③ 计分：faithfulness = 被支持的陈述数 / 总陈述数
```

**特性**：这是 RAGAS 最重要的指标。上下文里有但答案没说 → 分低（遗漏）；答案说了但上下文没有 → 分低（幻觉）。

------

### 2. Answer Relevancy（答案相关性）

**衡量的核心问题**：答案是否切题？是否回答了用户真正想问的？

**实现流程**：

```
① 反推问题：让 LLM 根据答案反向生成 N 个（通常 3 个）潜在问题
    答案是"交流电压，量程选750V" → 生成 ["万用表怎么测交流电？", "电压量程怎么选？", ...]

② 计算相似度：用 embedding 模型分别计算原问题和每个生成问题的向量
             计算原问题向量与每个生成问题向量的余弦相似度

③ 计分：answer_relevancy = 平均余弦相似度
```

**思想**：如果答案与问题相关，那么从答案反推出来的问题应该和原问题语义相近。如果答案答非所问，反推出来的问题会和原问题差很远。

**注意**：这个指标只看答案和问题的相关性，不看答案是否正确。

------

### 3. Context Precision（上下文精确度）

**衡量的核心问题**：检索出来的段落是否都相关？有没有召回一些不相关的内容来凑数？

**实现流程**：

```
① 逐段判断：按检索返回的排序，逐个判断每个段落是否与问题相关
    Prompt: "该文档是否与问题相关？只回答 true 或 false"

② 计算 precision@k：
    precision@k 只在位置 k 的文档相关时才计算
    公式：context_precision = Σ(precision@k × rel(k)) / 总相关文档数
    
    其中 precision@k = 前 k 个文档中相关文档数 / k
         rel(k) = 位置 k 的文档是否相关（0 或 1）
```

**含义**：越相关的文档排在越前面，分数越高。不相关的文档排前面会拉低分数。

**例**：检索到 3 个文档 [A, B, C]，实际 A 相关、B 不相关、C 相关：

- precision@1 = 1/1 = 1.0（A 相关）
- 不计算 precision@2（B 不相关）
- precision@3 = 2/3 ≈ 0.67（C 相关）
- score = (1.0 + 0.67) / 2 ≈ 0.83

------

### 4. Context Recall（上下文召回率）

**衡量的核心问题**：检索到的上下文是否覆盖了回答该问题所需的全部信息？

**实现流程**：

```
① 把标准答案（或 LLM 生成的最佳答案）拆成原子陈述

② 对每个陈述，判断是否可以从检索到的上下文中找到支持依据

③ context_recall = 被上下文支持的陈述数 / 总陈述数
```

**含义**：这个指标衡量检索的完整性——有没有漏掉关键信息。
